# 会话后复盘

## 概述

每次开发 session 结束时，对整段对话进行结构化回顾，分析四个维度，将经验教训写入 Claude Code 的记忆系统和 CLAUDE.md，实现跨会话经验复用。

**核心原则：** 模式胜过事件。单次失误不如反复出现的模式有价值。

**重要限制：** 复盘基于当前上下文中对 session 的回顾。Memory（记忆）可能不完整，尤其是长 session。聚焦你最确定的模式和最重要的教训，而不是追求全面。

## 何时使用

- **每次开发 session 结束时**运行
- 完成了某个功能、修复了某个 bug、或决定暂停工作
- 不建议 session 中间运行（数据不足以发现模式）

## 分析维度

### 维度 1：用户提示质量

分析用户（人类搭档）提问的方式是否有改进空间。

具体分析点：
- **清晰度：** 用户是否清晰表达了想要什么？还是需要多轮追问才能明确？
- **完整性：** 需求是否一次性完整提出？还是边做边改？
- **上下文：** 用户是否主动提供了文件路径、错误信息、预期结果等必要上下文？
- **反馈：** 用户是否在结果偏离期望时及时且明确地指出？

输出：评级（优秀/良好/一般/差）+ 1-2 条对用户有价值的改进建议

### 维度 2：Agent 思考过程

分析 Claude 在推理过程中的模式。

具体分析点：
- **过早结论：** 是否未经充分探索就锁定了方案？
- **未验证假设：** 是否有假设未被核实（如"假设 X 可以工作"但没验证）？
- **上下文遗漏：** 是否忽略了用户明确提供的信息？
- **探索平衡：** 是否在探索多种方案和深入实施之间取得了平衡？

输出：标记 1-2 个好/坏模式

### 维度 3：Agent 输出质量

分析 Claude 的输出是否符合预期。

具体分析点：
- **完整性：** 是否完全解决了用户的问题？还是留了占位符、TODO？
- **准确性：** 代码/方案是否经过了验证？是否有明显错误？
- **结构：** 输出的组织是否清晰、逻辑是否连贯？
- **验证：** 是否主动验证了结果（如运行测试、检查副作用）？

输出：评级（优秀/良好/一般/差）+ 具体改进点

### 维度 4：流程模式

分析整体协作中的过程性模式。

具体分析点：
- **目标匹配：** 双方是否在共同理解的目标上工作？还是有过误解？
- **重复劳动：** 是否有反复修改、来回确认浪费的精力？
- **成功模式：** 本 session 中什么做法特别高效？值得未来复制？
- **可自动化：** 有没有可以通过模板、checklist、配置来解决的重复性问题？

输出：最多 3 条最关键的经验教训

## 跨 session 复用的工作机制

1. 本技能将经验教训写入 `C:\Users\28634\CLAUDE.md`
2. Claude Code 在每个 session 启动时**自动加载** CLAUDE.md
3. 下次 session 开始时，经验教训已在上下文中
4. 多次复盘后，模式逐渐积累，形成对用户协作习惯的持续优化

不需要任何额外配置。这就是 CLAUDE.md 的固有功能。

## 常见错误

- ❌ **过度追求全面** — 如果 session 很长，不要试图回忆每一个细节。聚焦最重要的 3-5 个关键时刻
- ❌ **输出过于泛泛** — "提示要更清晰" 不如 "在提需求时附带具体文件路径" 
- ❌ **忽略正向反馈** — 也要记录什么做得好，下次继续保持
- ❌ **重复记录相同教训** — 先检查 CLAUDE.md 中已有的教训，合并而非重复
- ❌ **在 session 中间运行** — 数据不足时无法发现有效模式
